GENUI OPERATING STRATEGY

职路 AI GenUI 模块运行机制与调度策略

GenUI 承担元宝对话链路中的动态任务执行。系统结合当前任务、用户状态、会话进度与可信上下文，判断是否生成界面、选择交互形态、组织模型与工具执行，并将用户确认后的有效结果写回成长状态。具体功能共享同一套触发、编排、执行、提交、接力与降级机制。

核心闭环操作结果驱动下一步界面完成任务、状态保存结果、编排服务继续推进。
触发判断5 类信号操作需求、结构稳定性、状态价值、风险和端侧空间。
生命周期6 个阶段创建、草稿、校验、提交、接力、恢复。
共享底座5 套引擎文档、轨迹、测评、情境和证据状态。

降级、异常与可信运行

异常流程与正常流程使用同一状态规则：保留草稿、不提交不完整结果、向用户说明失败位置并提供可继续路径。

LOW RISK界面生成失败降级为结构化文本，保留重新生成轻量组件的入口。
MEDIUM RISK工具超时或失败保留草稿与已完成步骤，显示失败节点并支持重试或备用工具。
HIGH RISK来源或事实不足停止给出确定结论，区分已知事实、模型推断和待核实信息。
STATE RISK提交冲突通过版本与幂等键检测，展示差异并由用户选择合并方式。
异常场景用户界面系统动作状态规则监控信号
意图置信度不足轻量 Sheet 确认任务类型和目标暂停复杂 Artifact 生成不写长期状态任务澄清率、误触发率
Artifact Schema 校验失败展示结构化文本和“重新生成工具”记录 Schema 错误并切换安全模板保留会话，不产生状态补丁Schema 失败率、模板降级率
模型响应超时先显示文件结构、计算结果等确定性内容延后补充解释或切换轻量模型草稿持续可用首屏延迟、补全成功率
确定性工具失败标记失败步骤，允许重试或修改输入调用备用工具或返回可复现错误禁止提交不完整结果工具成功率、重试率
来源版本失效提示重新核验，不沿用旧结论刷新知识源与适用范围旧状态保留版本，不覆盖事实链来源过期率、复核触发率
用户中途关闭会话留下“继续上次任务”锚点保存 Session 草稿和当前步骤未提交内容不进入成长画像恢复率、草稿放弃率

客户端 GenUI Runtime

客户端负责安全渲染、即时操作、草稿保存与局部更新，复杂语义判断仍由后端完成。

模块主要职责运行策略关键约束
Artifact Registry根据 Artifact Type 选择 Renderer只加载已注册类型，按版本选择兼容渲染器禁止模型直接生成可执行 HTML
Component Library提供题目、时间轴、图谱、表格、拖拽和录音组件组件保持业务无关，领域语义由 Schema 注入组件参数必须通过 JSON Schema 校验
Local Draft Store保存未提交的排序、选择、输入和作答高频交互本地完成，定时生成轻量检查点草稿不得直接覆盖长期状态

Patch Renderer接收服务端 Patch 并局部更新界面优先更新受影响区域，避免整页重新生成Patch 只能修改 Schema 允许的路径
Session Recovery恢复中断任务和未完成草稿刷新或重新进入会话后恢复最近检查点过期任务需要重新校验上下文
Safe Renderer隔离未知组件、脚本与非法字段未知类型降级为结构化文本或静态卡片禁止任意脚本、外链执行和 DOM 注入

SHEETBottom Sheet参数配置、范围确认、风险提示。
FULLSCREEN临时全屏连续操作、多步骤任务、实验和模拟。
RESULT结果卡结果复盘、来源证据和下一步接力。

Artifact Service 与生命周期

Artifact Service 管理动态任务对象的创建、草稿、校验、提交、恢复和版本演进。

01Created初始化数据与可用动作
02Draft用户操作、本地草稿和撤销
03Validating规则、来源和工具校验
04Ready满足提交条件，等待确认
05Committed状态写回并发布接力事件
06Paused / Expired恢复、重新校验或重建
状态用户可见表现服务端处理长期状态影响异常恢复
Created显示初始摘要或骨架屏创建 Session、Schema 和允许动作无创建失败时返回文本说明
Draft可排序、输入、标记和撤销保存事件检查点，必要时执行轻量校验无刷新后恢复最近草稿
Validating显示校验进度和待处理项调用规则、工具与来源服务无失败步骤可单独重试
Ready显示提交预览和影响范围生成 State Patch 与证据列表尚未写入上下文变化时重新计算 Patch
Committed显示结果摘要与下一步原子写入状态并发布 Workflow Event正式写入幂等键防止重复提交
Paused / Expired显示继续任务或重新校验恢复 Session 或重新构造上下文保留已提交部分不静默沿用过期判断

模型与确定性工具路由

不同任务按复杂度、风险和价值选择端侧、规则、小模型、通用模型或人工复核。

L0端侧完成展开、拖动、筛选、参数预览、撤销和草稿缓存。
L1规则与工具时间求和、选项判定、图结构校验和格式检查。
L2小模型意图分类、字段提取、错因分类和风险标签。
L3通用大模型复杂解释、材料理解、追问、面试反馈和练习生成。
L4高质量模型 / 人工高风险专业材料、复杂主观评价和高价值复核。
目标路由策略关键措施
降低延迟前端和确定性规则优先骨架先返回，模型解释异步补充
降低成本小模型先分类，大模型按需调用缓存材料解析、批量生成练习和分层摘要
提高准确性计算、代码、判分走确定性工具模型输出不得覆盖工具结果
提高可信度结论绑定来源、版本和证据原始事实与模型推断分开呈现
控制风险高风险任务使用受控知识源来源不足时停止结论并转人工复核

State Committer 与长期状态

State Committer 是 Artifact 草稿与长期成长状态之间的提交闸门。

Artifact 草稿用户操作与未确认结果
→
字段校验完整性、范围和数据类型
→
证据校验来源、版本和推断标记
→
版本检查状态版本、冲突和幂等
→
Commit Preview展示影响范围并由用户确认
→
Durable State原子写入并触发接力
提交权限只有 Commit Event 可以修改长期状态。
事实优先用户确认内容优先于模型推断。
证据分离原始事实、工具结果与模型判断分开保存。
版本保留所有变化保留依据、版本与操作人。
幂等控制同一 Patch 不因重试重复写入。
冲突合并多端修改时展示差异，不静默覆盖。

数据存储与状态分层

会话、Artifact、长期成长状态、材料、向量索引和行为事件采用不同存储策略。

SECONDS–HOURS前端临时状态展开、排序、未提交答案和本地撤销。
HOURS–DAYSArtifact Session未完成计划、暂停模考、面试草稿和检查点。
LONG TERM成长状态目标、掌握度、错因、材料、任务和职业证据。
数据类型推荐存储主要原因保留策略
会话消息文档数据库适合按会话读取和多类型消息结构按用户策略归档或删除
活跃 ArtifactRedis + 文档数据库兼顾低延迟、恢复与历史版本活跃缓存过期后转长期存档
成长状态关系型数据库需要事务、版本和结构化查询持续保存，可审计和回滚
材料与附件对象存储保存 PDF、图片、音频和 Office 文件按用户授权和生命周期管理
文本切片与检索向量数据库 / 搜索引擎支持材料与授权知识源检索材料删除时同步清理索引
事件日志消息队列 + 数据仓库异步分析、漏斗和质量监控去标识化后按合规周期保留
模型调用记录日志与指标系统分析延迟、成本、错误与质量敏感上下文脱敏后记录

可靠性与降级设计

每类故障都需要保留用户草稿、明确失败位置，并避免写入不完整状态。

异常用户侧表现系统处理状态保护
模型超时解释仍在生成先展示工具结果和结构，解释异步补充不影响当前草稿
工具失败某一步无法计算显示失败步骤、允许重试或切换备用工具不提交未验证结果
Schema 不合法界面无法安全渲染降级为结构化文本或静态表格保留任务上下文
状态版本冲突其他设备已修改计划展示差异、生成可合并 Patch 并重新确认禁止静默覆盖
来源失效引用无法打开标记失效并阻断高风险结论提交保留已验证内容
客户端版本过低不支持新组件选择兼容 Renderer 或文本降级保持事件协议向后兼容
网络中断操作暂时无法同步本地缓存事件，恢复后按序补交使用幂等键去重
Artifact 过期上下文可能已经变化重新获取上下文并再次校验保留已提交部分

安全、隐私与权限控制

简历、讲义、语音、学习记录和职业证据按数据对象和使用目的实施最小权限控制。

文件安全上传前进行类型、病毒和恶意内容检查。
最小上下文模型请求只携带当前任务所需字段。

日志脱敏身份、简历、材料内容和联系方式脱敏。
数据控制用户可以删除材料、状态和历史记录。
审计追踪高风险工具和状态提交保留完整审计链。
数据层级示例访问主体使用规则
私有原始数据简历、讲义、语音用户本人和授权服务对象级权限，默认不用于公共训练

延迟与成本策略

以“首次可操作”和“单次有效状态写回成本”为核心优化目标。

≤ 300ms触发判断规则与小模型完成任务分类。
≤ 800msArtifact 骨架模板和缓存优先返回。
≤ 1.5s首屏可操作先展示确定性内容和加载状态。
≤ 500ms工具反馈计算、判分和本地验证。
≤ 3s模型解释异步补充复杂说明和反馈。
≤ 800ms状态提交校验、事务写入和接力事件。

成本控制组合

小模型负责意图和字段分类；Artifact 骨架由模板与规则生成；材料解析与知识检索结果缓存；练习题采用批量预生成；长内容使用分层摘要；高成本模型仅用于复杂规划、高价值反馈和高风险复核。

部署与演进路径

架构从单体编排逐步演进到可供元宝多个业务复用的 GenUI 平台。

PHASE 1单体编排Gateway、Workflow、Artifact Service、Redis 与关系库组成最小闭环，支持原型验证和小规模上线。
  • 单一模型路由
  • 少量工具服务
  • 统一状态协议
PHASE 2服务拆分按文件、测评、情境、状态和模型路由拆分独立服务，引入消息队列与事件仓库。
  • 领域服务独立扩容
  • 灰度和熔断能力
  • 统一可观测性
PHASE 3平台化能力形成通用 Artifact Registry、Workflow DSL、专业知识源平台和多业务租户隔离。
  • 组件与流程市场
  • 模型成本调度
  • 安全审核中心

技术架构结论

三项核心能力共同形成对话原生的任务执行闭环。

01动态任务编排将自然语言、材料和用户状态转化为受约束、可执行的 Artifact。
02确定执行与可信生成模型负责理解和解释，工具负责计算、验证、来源与风险约束。
03状态驱动的连续服务把用户确认结果沉淀为长期状态，并据此生成下一项任务。
对话理解 → 动态界面 → 用户操作 → 工具验证 → 状态提交 → 下一任务编排
